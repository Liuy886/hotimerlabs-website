# Web3 Agency 竞品研究与 HotimerLabs 优化方向

研究对象：Coinbound、MarketAcross、Lunar Strategy、Crowdcreate、Avark、Generis、SHILLR，以及若干 X/Twitter 主页结果。

## 1. 竞品共性

### 1. 首屏必须非常快地建立“我是谁 + 我能带来什么结果”

强竞品不会在首屏解释太多概念，而是直接打出：
- Crypto / Web3 marketing agency
- Blockchain PR agency
- Web3 design and branding agency
- Consulting & growth agency in Web3

HotimerLabs 应该保留清晰业务关键词：
- Growth
- Listing
- Liquidity
- GTM
- MM coordination
- Exchange-side resources

### 2. 可信背书比炫酷视觉更重要

成熟机构通常会展示：
- 服务过多少客户
- 多少年行业经验
- 多少 Campaign / PR / case
- 客户 Logo
- 媒体露出
- 具体案例

HotimerLabs 当前不能公开客户，所以需要用“匿名委托场景”替代：
- Listing readiness mandate
- Regional GTM mandate
- Liquidity coordination mandate
- Exchange growth mandate

重点是让用户相信我们处理过真实、高敏感度、高价值事项。

### 3. 服务最好按项目生命周期组织

Generis 的结构很适合借鉴：Development / Launch / Scaling / Custom Solution。

HotimerLabs 可以改成：
- Incubation：定位、叙事、融资、路线图
- Launch：GTM、KOL、社区、PR、Listing 准备
- Listing：交易所匹配、材料、节奏、沟通
- Post-listing：MM、流动性、市场维护、复盘
- Exchange growth：项目资源、活动、区域增长

这样比单纯罗列服务更像咨询公司。

### 4. Web3 Agency 的 X/Twitter 表达要有“运营中”的感觉

观察到的常见做法：
- 主页 bio 非常短，直接说明服务对象和核心能力
- 置顶内容通常是案例、活动、观点或服务入口
- 经常发布行业观点，而不只是公司广告
- 使用 Telegram / Linktree / 官网作为转化入口
- 有些机构会用 X Spaces、AMA、活动、客户公告来体现资源网络

HotimerLabs 后续 X 内容方向：
- Listing readiness checklist
- GTM mistakes for token projects
- Exchange-side project screening notes
- MM / liquidity education
- Anonymous market notes
- Founder advisory threads

### 5. 视觉风格分两派

专业机构型：
- 深色、留白、Logo 墙、数据指标、案例、表单
- 适合 Listing、PR、交易所、基金、机构客户

Web3 原生型：
- 更强对比、更锐利的网格、动态感、技术感
- 适合项目方、早期团队、生态活动、GTM Campaign

HotimerLabs 已有双模式切换，这个方向是对的。但首页默认应该更稳重，Native 模式作为加分项。

## 2. 值得借鉴的页面模块

### 模块 A：Lifecycle Services

把服务从“服务清单”升级为“项目阶段方案”：
- Build the narrative
- Prepare the market
- Secure listing readiness
- Coordinate liquidity
- Scale exchange and ecosystem growth

### 模块 B：Anonymous Mandates

不能公开客户时，用委托类型和结果导向写法：
- Listing Preparation Mandate
- Regional GTM Mandate
- Liquidity Coordination Mandate

### 模块 C：Operating Principles

强调我们不是短期刷量团队：
- Confidential by default
- Strategy before noise
- Exchange-side and project-side understanding
- Execution with feedback loops

### 模块 D：Direct Telegram CTA

Web3 行业里 Telegram 是自然入口。官网表单可以保留，但主 CTA 应该始终能快速打开 Telegram。

## 3. HotimerLabs 立即优化方向

1. 首页服务区改成“Lifecycle”表达，而不是普通 5 张卡片。
2. 增加“匿名委托场景”模块，强化真实业务感。
3. 增加“Operating Principles”模块，区分低质量营销 Agency。
4. 收紧首屏字号和高度，让页面更高级。
5. 保留双语与双风格，但默认视觉更偏机构型。
6. 后续需要建立 X/Twitter 内容阵地，与官网形成闭环。
