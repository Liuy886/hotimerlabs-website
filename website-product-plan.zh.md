# Web3 咨询公司官网产品方案

## 1. 产品定位

为一家服务交易所、协议项目、Token 项目和生态基金的 Web3 咨询公司搭建一个专业商务官网。网站需要快速建立信任，清晰解释服务能力，并将合格潜在客户转化为私密咨询沟通。

核心定位：
- 一家覆盖 Web3 增长与资本市场服务全周期的合作伙伴。
- 擅长项目孵化、市场进入、交易所 Listing、Market Making 协调、社区增长和战略咨询。
- 适合早期项目、成熟 Token 项目、寻求生态扩张的交易所，以及进入 Web3 的机构。

建议中文定位语：
> Web3 项目的增长、上市与市场流动性战略伙伴

建议英文定位语：
> Strategic Growth, Listing and Liquidity Partner for Web3 Projects

## 2. 目标客户

主要受众：
- Web3 项目创始人和核心团队
- 正在准备 Listing 或市场扩张的 Token 项目
- 寻求优质项目资源和市场活动的交易所
- 需要亚洲或全球市场资源的 VC-backed 加密创业公司
- 探索 Web3 转型的传统企业

关键客户顾虑：
- 你们是否真的拥有交易所、KOL、社区和流动性资源？
- 你们是否能设计一条从产品到市场的可信路线图？
- 你们是否能避免低质量流量和短期噪音？
- 你们是否能支持保密性高、风险敏感的 Listing 和 MM 工作？
- 你们是否同时理解项目方和交易所的业务逻辑？

## 3. 网站目标

业务目标：
- 获取高质量咨询线索。
- 以清晰且高级的方式展示服务范围。
- 在正式销售沟通前建立可信度。
- 将复杂的 Web3 服务包装成容易理解的服务模块。
- 在后续版本中支持中文和国际访客。

转化目标：
- 主要 CTA：预约咨询 / Book a Consultation
- 次要 CTA：通过 Telegram / Email / WhatsApp 联系
- 第三级 CTA：下载公司介绍或提交项目信息

## 4. 推荐网站结构

第一版建议做一个聚焦的单页网站，并配合清晰的分区导航。这样可以快速开发，同时保持信息表达的控制力。

页面模块：
1. 首屏 Hero
   - 清晰定位
   - 一个强价值主张
   - 主要 CTA
   - 信任提示，例如服务类别或资源覆盖范围

2. 我们服务谁
   - 项目团队
   - 交易所
   - 基金和生态
   - 进入 Web3 的传统企业

3. 核心服务
   - 孵化与战略
   - 市场扩张与营销 Campaign
   - Listing 咨询
   - Market Making 协调
   - 社区与 KOL 增长
   - 交易所与生态合作

4. 合作模式
   - 诊断
   - 策略设计
   - 资源匹配
   - 执行管理
   - 复盘与优化

5. 为什么选择我们
   - Web3 原生经验
   - 同时理解交易所端和项目端
   - 具备增长、Listing 与流动性的综合视角
   - 保密执行
   - 全球与亚洲市场资源

6. 案例 / 过往经验占位
   - 在 V1 中，如果无法展示公开 Logo，可以使用匿名化能力背书。
   - 示例：“支持多个 Token 项目的 Listing 准备”、“搭建区域市场 GTM 活动”、“协调 KOL 与社区增长。”

7. 联系 / 线索表单
   - 项目名称
   - 官网或 Deck 链接
   - 当前阶段
   - 所需服务
   - Telegram 或邮箱
   - 留言

## 5. 服务包装

建议对外展示的服务名称：

### 项目孵化
定位、Token 叙事、路线图规划、生态策略、融资支持和资源规划。

### 市场进入
市场进入策略、社区增长、KOL Campaign、PR 规划、大使体系和区域扩张。

### Listing 咨询
交易所匹配、Listing 准备度评估、申请材料准备、市场节奏规划和沟通支持。

### Market Making 协调
流动性策略、MM 合作方匹配、上市后表现复盘和市场结构咨询。

### 交易所增长支持
项目资源挖掘、活动策划、区域合作、生态扩张和用户增长计划。

## 6. 品牌调性

网站应呈现：
- 高级但不浮夸
- 国际化但务实
- 可信、审慎、资源驱动
- Web3 原生，但不过度使用黑话
- 更像一家精品战略咨询机构，而不是喧闹的营销 Agency

视觉方向：
- 深色专业基底，搭配高对比度文字
- 克制使用金属色、青色或绿色强调色
- 使用数据网格 / 市场结构类视觉语言
- 避免 Meme 感过重、卡通化或过度投机感的风格

## 7. 核心信息草案

首屏标题选项：
- Web3 项目的增长、上市与流动性战略伙伴
- 从孵化到上市，帮助 Web3 项目进入真实市场
- 为交易所与项目方提供增长、Listing 与 MM 支持

首屏辅助文案：
> 我们为 Web3 项目方、交易所与生态机构提供从战略定位、市场推广、社区增长、Listing 策略到 MM 协调的一站式咨询与执行支持，帮助项目更高效地完成市场进入、资源匹配与长期增长。

主要 CTA：
- 预约战略咨询

次要 CTA：
- 查看服务范围

## 8. MVP 范围

V1 交付内容：
- 响应式单页网站
- 中文文案优先
- 预留英文结构
- 联系表单 UI
- Telegram / Email CTA
- 基础 SEO 元数据
- 移动端友好布局
- 隐私友好的线索收集说明

V1 暂不包含：
- 博客系统
- 客户门户
- CMS
- 复杂动画
- 需要法务批准的公开案例
- 支付或账户系统

## 9. 执行计划

阶段 1：产品与内容
- 确认公司名称、Logo 状态和联系渠道。
- 敲定服务名称和首页文案。
- 决定是否展示公开合作伙伴 / 客户 Logo，或使用匿名化能力背书。

阶段 2：设计与前端
- 搭建一个精致的响应式 Landing Page。
- 创建服务卡片、合作流程、能力背书和线索表单。
- 加入强识别度的首屏视觉身份。

阶段 3：转化与信任
- 增加表单校验。
- 增加 Telegram / 邮箱直接联系入口。
- 如有需要，增加可下载的公司介绍。
- 部署后增加数据分析。

阶段 4：上线
- 部署到 Vercel、Cloudflare Pages 或类似平台。
- 绑定域名。
- 配置 SEO 标题、描述、Open Graph 图片和 Favicon。

## 10. 需要创始人提供的信息

最终文案前至少需要：
- 公司名称
- 偏好语言：仅中文、仅英文或中英双语
- 公开联系方式：Telegram、邮箱、WhatsApp、表单或 Calendly
- 是否有公开客户、合作伙伴或匿名化成果
- 地域重点：亚洲、全球、中东、欧洲或具体市场
- 品牌偏好：保守机构风格，还是更锋利的 Web3 原生风格

如果暂未确定，建议默认方案：
- 使用临时品牌名，并搭建灵活结构。
- 先上线中文版。
- 合作伙伴 / 客户相关表述保持匿名化。
- 使用 Telegram 和邮箱作为主要线索渠道。
